MySQL (SQL)

medicines Table

CREATE TABLE medicines (
  id INT PRIMARY KEY,
  name VARCHAR(255),
  dosage VARCHAR(100),
  frequency INT,
  times JSON,
  notes TEXT,
  user_id INT,
  created_at TIMESTAMP,
  is_active BOOLEAN
);

medicine_history Table

CREATE TABLE medicine_history (
  id INT PRIMARY KEY,
  medicine_id INT,
  taken_at TIMESTAMP,
  scheduled_time TIME,
  status VARCHAR(50),
  user_id INT
);

function checkReminders() {
    const now = new Date();
    medicines.forEach(medicine => {
        medicine.times.forEach(time => {
            if (isTimeMatch(now, time)) {
                showNotification(medicine);
            }
        });
    });
}

API Endpoints

Medicine Management APIs

POST /api/medicines Add a new medicine
GET /api/medicines Fetch all medicines
PUT /api/medicines/:id Update a medicine
DELETE /api/medicines/:id Delete a medicine

History Management APIs

POST /api/history Record medicine intake
GET /api/history Fetch medication history
GET /api/history/today Get today's history

Sample API Request:

Challenges & Solutions

Challenge: Browser Notification Permissions

Solution: Implemented graceful fallback with visual alerts and audio notifications when browser permissions are denied.

Challenge: Time Zone Handling

Solution: Used JavaScript's built-in Date object with local time zone support for accurate reminder timing.

Challenge: Data Persistence

Solution: Implemented localStorage for client-side storage with plans for database integration.

Challenge: Cross-Device Synchronization

Solution: Designed RESTful API architecture to enable future cloud synchronization capabilities.

Challenge: User Interface Complexity

Solution: Adopted progressive disclosure design pattern to show relevant information contextually.

Challenge: Mobile Responsiveness

Solution: Implemented mobile-first CSS approach with flexible grid layouts and touch-friendly interfaces.
